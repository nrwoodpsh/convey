# 03. publishing 아키텍처

승인된 완성본(쇼츠 mp4)을 외부 채널로 배포하고 발행 이력을 관리한다. 1차 채널은 YouTube Shorts. **커모디티(외주·얇게)** — YouTube Data API 래퍼 최소 구현(알파 아님).

## 구성

| 서비스 | 배포형태 | 역할 |
|:---|:---|:---|
| `publishing` | API | 발행 요청·승인, YouTube 업로드, 발행 이력 |

## 흐름

```
content.approved
   ▼
[publishing]  PublishRequest(사람 승인 확인)
   │ YouTube Data API 업로드(OAuth)   ← external_client.py
   ▼
content.published
```

## 데이터

- **publishing_db**: Channel(YouTube 등), PublishRequest(승인), Publication(발행 이력·플랫폼 ID)

## 이벤트

- **구독**: `content.approved`
- **발행**: `content.published`

## 외부 연동

- YouTube Data API(업로드) — `external_client.py` 부패방지 계층. OAuth 토큰은 `.env`.
- 향후 Notion/SNS 채널 추가 가능(Notion MCP 승인 시).

## 가드레일

- **자동 발행 금지** — `PublishRequest` 승인(`content.approved`) 없이는 업로드하지 않는다.
