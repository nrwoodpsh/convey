# 01. research 아키텍처

시세·뉴스를 **종목·사건 지식 그래프**로 구조화해 저장한다(알파1: 근거·정확). 콘텐츠 제작에는 **관계·인과가 담긴** 근거 데이터를 공급한다. LoRA(모델 미세조정 기법)를 안 쓰므로 이 지식 그래프가 품질의 유일한 레버다(ADR 0005).

## 구성

| 서비스 | 배포형태 | 역할 |
|:---|:---|:---|
| `market-feed` | 워커 | KIS 시세 수집 → Kafka (`market.ticks`) — 이슈감지·차트데이터 소스 |
| `news-feed` | 워커 | 뉴스·RSS 수집 + **종목·사건 태깅** → Kafka (`research.ingested`) |
| `research` | API | 추출 → **Neo4j 지식 그래프**(관계·인과) + **Postgres**(시세·기사 사실) + `/search`(GraphRAG+SQL) |

> `news-feed`는 원형 `research-feed`를 개명·확장(태깅 추가). `market-feed`는 원형 그대로 실사용.

## 흐름

```
KIS 시세 ── market-feed ──[ market.ticks ]──┐
뉴스·RSS ── news-feed(종목·사건 태깅) ─[ research.ingested ]─┐
                                                            ▼
                    [research]  추출(NER+관계추출) →
                       ├─ Neo4j (research_graph): 노드·엣지 = 관계·인과
                       └─ Postgres (research_db): 시세·기사 사실
                                              ⇒ 축적 완료 (콘텐츠 자동 유발 X)
```

> **진짜 일 = 추출 파이프라인**: 뉴스 → 엔티티(NER) + 관계추출 → 그래프 엣지. 여기가 품질의 원천.
> 콘텐츠 생성 시 agent가 `/search`로 그래프 traversal(관계) + SQL 사실 회수를 pull(**GraphRAG** — 벡터 아님).

## 데이터 (하이브리드 — `research`가 두 스토어 소유)

**research_graph (Neo4j) — 관계·인과 [알파]**
- 노드(안): `Stock`(종목) · `Event`(사건) · `Sector`(섹터) · `Company/Person`
- 엣지(안): `HAS_EVENT` · `AFFECTS`(인과) · `SUPPLIES`(공급망) · `COMPETES` · `BELONGS_TO`
- 용도: 다홉 traversal — "삼성 실적↑ → 협력사 수혜 → 섹터 파급" 같은 관계 사슬

**research_db (Postgres) — 사실**
- `PriceTick`(시세 시계열 — 차트·이슈감지), `Article`(원문+출처/라이선스+발행일)
- 정확 조회 = SQL 인덱스(ticker·date). **벡터 의미검색(pgvector)은 POC 제외**(ADR 0006).

> 그래프 노드는 Postgres 사실(기사·시세)과 ID로 연결. 스키마·엣지 타입은 `/design` 확정.

## 이벤트

- **발행**: `market.ticks`(market-feed), `research.ingested`(news-feed, 축적 완료 — 자동 체이닝 없음)
- **구독**: 없음(파이프라인 시작점). issue-detector가 위 두 토픽을 구독.

## API

- `GET /research/search` — agent 근거 회수(GraphRAG: 관계 traversal + 종목·기간 SQL 사실). east-west 호출.
- (조회용) 종목·사건·기사 조회 엔드포인트 — /design 확정.

## 외부 연동

- KIS OpenAPI(시세), 뉴스·RSS 피드 — `external_client.py` 부패방지 계층.
- 텍스트 추론은 로컬 Ollama만(외부 텍스트 LLM 금지).

## 가드레일

- 원문·기사는 **출처 URL·라이선스 메타 필수 동반**(무출처 생성 금지·알파1의 근거).
