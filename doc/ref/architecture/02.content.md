# 02. content 아키텍처

이슈 종목을 키워드로 **근거에 못박은 스크립트**를 만들고, 미디어를 붙여 쇼츠로 합성한다. 생성 잡·스크립트·자산·완성본 상태를 소유(별도 media 도메인 없음). **알파는 근거 스크립트(1)·정확 렌더(3)·양산 안정성(4)** — broll(배경 영상 클립)·TTS·스티칭 같은 커모디티는 외부 API로 얇게.

## 구성

| 서비스 | 배포형태 | 역할 | 알파/커모디티 |
|:---|:---|:---|:---|
| `content` | API | 생성 잡·단계 상태, 스크립트·자산·완성본, 승인, 히스토리 | ★알파4 |
| `agent` | API | RAG 2곳 검색 → **엄격 인용 스크립트** 오케스트레이션 | ★알파1 |
| `llm-inference` | API | 로컬 Ollama 창구 | — |
| `image-gen` · `tts` | API/워커 | broll 이미지·TTS 음성 (외부 API 래퍼) | (외주) |
| `video-assembly` | 워커 | **정확한 차트·수치 오버레이 + ffmpeg 합성 → mp4** | ★알파3 |

## 흐름

트리거는 **이슈 종목/주제 기반**(사람 `POST /content/generate`; 이슈는 issue-detector가 추천). 리서치 원문 도착은 자동 유발 안 함.

```
이슈 종목(키워드) ──► POST /content/generate { 종목/주제 }
   ▼
[content] 잡 큐잉 → content.generate(자기 발행) → consumer 픽업
   │ ★agent ─ GET /research/search + GET /content/search (RAG 2곳)
   │          → 근거 수집(수치·출처) → 엄격 인용 스크립트 → llm-inference(로컬)
   ▼
(외주) image.generate · tts.generate   (외부 API: broll·음성)
   │            │
image-gen     tts
   ▼            ▼
   └── 자산 완료 ──┘
   ▼
★[video-assembly]  정확한 차트·종가·등락률 오버레이 + broll + 음성 + 자막
                    → ffmpeg 합성 → mp4  (내부 status=ready)
   ▼
[content] 완성본 저장 → 【사람 승인】 ─► content.approved
```

> **알파의 핵심**: 생성형 영상 모델은 화면 위 텍스트·숫자를 "그리는" 것이라 정확한 종가·등락률·한글 자막을 못 박는다(프레임마다 뭉개짐). 그래서 **화면 위 정확한 수치·차트는 `video-assembly`가 결정론적으로(matplotlib/ffmpeg drawtext) 렌더·합성** — 검증 가능한 정확성이 우리 통제 영역. 외부는 **배경 broll·음성(TTS)** 등 "느낌"만 담당(커모디티). TTS 내레이션 속 숫자는 텍스트를 읽는 것이라 안전.
> **POC 범위**: 생성형 영상 모델은 POC에서 제외. 배경(스톡 영상/모션) + 정확 차트 애니 + 텍스트 오버레이 + TTS로 시작 = 100% 정확·저복잡도. 생성형 영상은 후속 커모디티로 얇게(ADR 0006).

## 데이터

- **content_db**: GenerationJob(단계 상태), Script(스크립트 + **인용/출처 근거**), Asset(broll·음성 메타), Content(완성본 메타), ReviewStatus. 히스토리 중복회피는 키워드/메타 조회(벡터 아님 — ADR 0006).
- ⚠️ **미결(미디어 라운드)**: 미디어·mp4 **바이너리 파일** 저장소 미정 — 오브젝트 스토리지(MinIO/S3) vs 로컬 볼륨. content_db엔 메타·경로만.

## 이벤트

- **발행(자기 큐)**: `content.generate` — API가 잡 큐잉 → consumer 비동기 픽업
- **구독**: `content.generate`, (미디어 완료 신호)
- **발행(fan-out)**: `image.generate`·`tts.generate` (커모디티 미디어)
- **발행(승인)**: `content.approved` (합성 완료는 이벤트 아님 — 내부 status=ready)
- fan-out/join·양산 재개(잡 상태 기반) 방식은 `/design` 확정.

## 외부 연동

- broll 이미지·TTS = 외부 API(부패방지 계층). **원문 전체 반출 금지 — 프롬프트·에셋만.**
- 텍스트 생성·차트 렌더 = 로컬(Ollama / ffmpeg).

## 가드레일

- 스크립트의 **모든 수치는 출처 있는 데이터에 못박음**(환각 금지 = 알파1). 무출처 생성 금지.
- 완성본 발행은 **사람 승인 후에만** → `publishing`.
