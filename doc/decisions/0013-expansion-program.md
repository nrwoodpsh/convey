# ADR 0013 — 확장 프로그램: 배경 컷 전환·종목 마스터·수집 확대·안정화 (C 제외)

- **상태**: 채택 (2026-07-23)
- **맥락**: 파이프라인·품질·연출·성능이 한 사이클 정리된 뒤, 배포·인증·YouTube 자동(=C)을 제외한 남은 개선을 한 번에 진행하기로 했다. 항목: 연출 심화(배경 컷 전환·수치 팝), 데이터 확장(종목 마스터·수집 소스·과거 기사 관계 백필), 안정화(재시도·자동양산 검증·모니터링).
- **결정**:
  - **배경 컷/xfade 전환 채택** — 세그먼트별 다른 Pexels 클립을 xfade로 전환(최대 3개), 실패 시 단일 배경 폴백. **㉖(ADR 없음, task-direction)에서 "안정 위해 단일 배경"으로 미뤘던 것을 대체**한다(사용자가 화려함을 명시 선택). 수치는 chart 구간에 scale 바운스 팝.
  - **종목 마스터는 오프라인 pykrx 생성→커밋**(정적). 런타임 pykrx 의존·비결정론 회피. `scripts/gen-stocks.py`가 `common/stocks.py`를 재생성(사람 실행).
  - **DART는 재수집이 아니라 활용 강화** — 이미 news-feed가 공시를 수집 중 → 공시 유형을 그래프 사건(HAS_EVENT)으로 태깅. 수집 소스는 시세 종목·뉴스 RSS를 넓힌다.
  - **과거 기사 LLM 관계 백필** — 현행 백필(결정론 섹터 엣지)에 `--llm` 옵션으로 과거 본문 관계추출(Ollama, 일회성·재개 가능).
  - **재시도는 content 소비자에서 멱등·상한(2)** — Kafka DLQ/재시도 토픽은 POC 과함, 최소 변경. 모니터링은 `/ui/stats`(로컬 무인증)로 충분(Prometheus 등은 배포=C 영역).
- **트레이드오프**: 배경 컷은 렌더 시간·실패 위험↑(폴백으로 방어). 종목 마스터 정적 생성은 최신성↓(주기적 재생성 필요). 백필은 Ollama 부하(일회성). 전 항목 **가드레일 불변**(수치=사실·무출처 금지·발행 사람 승인).
- **대안(기각)**: 단일 배경 유지(㉖) → 사용자 거부. 런타임 종목 로드 → 배포 의존. DART 신규 수집기 → 이미 존재. Kafka DLQ → POC 과함.
- **영향**: video-assembly(멀티 broll·xfade·scale 팝), content(broll_queries·재시도·/ui/stats), common(종목 마스터), market-feed·news-feed(범위), research(HAS_EVENT·백필 --llm). 구현은 Phase D→E→F. **C(배포·YouTube·Supabase)는 범위 밖**(backlog-deferred).
- **관련**: [0004](알파), [0005](그래프 관계), [0008](데이터 소스), [0009](broll 무료 스톡), [0012](품질 — 그래프 인과·LLM 편집자). ㉖의 단일 배경 방침을 대체.
