# ADR 0012 — 품질: 그래프 인과 투입 + LLM은 편집자(연결자), 정확성 불가침

- **상태**: 채택 (2026-07-23)
- **맥락**: 1차 구현 후 영상 품질이 "헤드라인 원문 + 종가/등락 + 거시 숫자 덤프"의 나열이라 제품화가 어렵다. 코드 추적 결과 **알파("그날 기사 × 그래프 인과")가 파이프라인에서 끊겨 있었다** — agent `Evidence`에 `relations` 필드가 없어 research `/search`가 반환한 그래프 관계(RelationHit)를 버리고 있었고, `build_script`는 훅 한 문장만 LLM으로 쓰고 나머지는 헤드라인·숫자를 붙여넣었다. 정확성 알파 때문에 LLM에 자유 창작을 시킬 수 없어, 품질을 어디서 낼지 원칙이 필요하다.
- **결정**:
  - **품질의 3원천**을 명시한다: ① 더 좋은 데이터(그래프 인과·사실 큐레이션), ② LLM은 **연결자/편집자**(주어진 사실·관계만 잇는 문장), ③ 연출(장면·자막 싱크·배지). 창작으로 품질을 내지 않는다.
  - **그래프 인과를 스크립트까지 흘린다**: agent `Evidence.relations` 부활 → `build_script`가 `relation` 섹션 생성(근거 결속). 그래프는 확대·백필로 채운다(엔티티 사전↑ + 단일 엔티티 엣지).
  - **정확성 불가침**: 모든 수치는 price/macro 슬롯(사실), 모든 관계는 그래프(근거 article_id). LLM은 문장만 쓰고 값·새 주장은 만들지 않는다. **인용 검증(citation guard)**을 유지해 위반 문장은 폐기·fallback.
  - 구현은 **Phase A(데이터)→B(문장)→C(연출)** 단계별. A만으로도 인과가 생겨 헤드라인 리더에서 벗어난다.
- **트레이드오프**: LLM 연결자는 문장 유창성과 정확성의 긴장을 낳음 → 프롬프트 제약 + citation guard로 방어(유창성 일부 희생). 그래프 백필·연출은 렌더/처리 비용↑ → 세그먼트 수 제한·안전 강등.
- **대안(기각)**: LLM 자유 서술(환각·알파 위배) / research가 문장 생성(도메인 경계 위반) / 벡터 RAG 도입(ADR 0006 제외 유지) / 연출을 유료 생성형 영상으로(ADR 0009·0006 위배).
- **영향**: agent(retriever·builder), research(/search 큐레이션·그래프 확대·백필), news-feed·common(엔티티 사전), content·video-assembly(연출 데이터·구간 렌더·배지·TTS). 계약 `api-contract-quality.py`. 신규 외부 엔드포인트 없음(내부 개선 + 일회성 백필).
- **관련**: [0004](알파1 정확 정보·알파3 정확 렌더), [0005](GraphRAG — 관계·인과의 근거), [0006](벡터 제외·POC 범위), [0011](대시보드 승인·수정 — 시나리오 편집이 여기에 얹힘). ㉑~㉔의 표면 개선을 데이터·문장·연출로 심화.
