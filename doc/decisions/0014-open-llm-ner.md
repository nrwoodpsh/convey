# ADR 0014 — 엔티티 인식: 로컬 Ollama 개방형 추출 + 수집 멱등

- **상태**: 채택 (2026-07-23)
- **맥락**: 지식 그래프가 46개 종목 사전에 갇혀 노드 33·관계 69에 불과(기사 1,085건 대비 빈약). 사전 밖 기업·인물·사건·인과가 그래프에 안 들어와 GraphRAG(알파)가 얕다. 더 많은 기사를 "넣는" 것이 아니라(모든 기사는 이미 Postgres에 저장) **인식되는 엔티티 범위**가 병목.
- **결정**:
  - **개방형 NER를 로컬 Ollama로** — 기사 본문에서 엔티티(기업·인물·사건·기관·섹터)와 관계를 **한 번의 LLM 호출**로 추출(기존 `extract_relations` 확장). 사전(고정확 seed)과 **합집**.
  - **가드레일 불변**: 추출 엔티티는 **본문 부분문자열**이어야 채택(LLM 환각 엔티티 폐기), 관계는 **근거 article_id 결속**·엣지 어휘 제한, 정규화·스톱워드로 노이즈 차단. 수치는 만들지 않음(NER는 텍스트 엔티티만).
  - **텍스트 LLM은 로컬 Ollama만**(가드레일) — 통계 NER 모델(spaCy/KoNLPy)은 새 ML 의존·다운로드·한국어 편차로 기각. 기존 llm-inference 재사용·1콜 효율.
  - **추출 위치 = research consumer**(그래프·LLM 소유). news-feed는 사전 seed만 계속 제공.
  - **수집 멱등 도입**: news-feed가 매 폴링 사이클 같은 기사를 재발행 → 지금은 중복 저장·재추출. NER는 비싸므로 **source_url 기준 신규 기사만 저장·추출**(멱등). 라이브는 신규만, 과거는 일회성 전량 백필(1,085).
- **트레이드오프**: 기사당 Ollama 1콜 → 부하↑(멱등으로 1회만·백필은 일회성으로 완화). 개방 추출은 노이즈·중복 노드 위험 → substring 검증·정규화·스톱워드로 방어(완전한 entity linking은 후속). 유창성보다 정확·근거 우선.
- **대안(기각)**: 통계 NER 모델(의존·품질) / 원문 통째 그래프 적재(노이즈, GraphRAG는 관계 구조를 탐색) / news-feed에 LLM 추가(결합).
- **영향**: research `extract`(extract_graph)·`consumer`(멱등·개방 추출)·`backfill`(전량). 그래프 노드·관계 대폭 증가 기대. 계약 `api-contract-ner.py`. 벡터 임베딩은 여전히 제외(ADR 0006).
- **관련**: [0005](GraphRAG — 관계·인과), [0012](품질 — 그래프 인과·LLM 편집자), [0006](벡터 제외·POC), [0004](알파1 정확·근거). ㉕/A3(결정론 섹터 엣지)·㉙/E3(HAS_EVENT)를 개방형으로 확장.
