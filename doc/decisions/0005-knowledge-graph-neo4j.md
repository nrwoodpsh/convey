# ADR 0005 — 지식 레이어를 그래프(Neo4j)로. [0002]·[0004]의 "그래프 미사용" 대체

- **상태**: 채택 (2026-07-18) — [0002]·[0004]의 "그래프 DB 미사용" 결정을 **대체**
- **맥락**: LoRA 제거([0004])로 콘텐츠 품질·차별성의 유일한 레버는 **지식 레이어**(무엇을 어떻게 구조화해 넣는가). 금융 콘텐츠의 substance는 종목·사건 간 **관계·인과**(수혜·경쟁·섹터 파급)다. 벡터 유사도나 관계형 조인으로는 다홉 관계 추론이 약하다. 또 나중에 엔진을 이관하면 `research`의 저장/쿼리 코드를 재작업해야 한다 → **처음부터 목표 엔진**으로 간다.
- **결정**: 지식 레이어를 **Neo4j(전용 그래프 DB)**로 처음부터 구축한다.
  - `research`가 **두 스토어 소유**:
    - `research_db` (Postgres + pgvector): 시세 시계열 · 기사 원문 · 출처/라이선스 · 텍스트 임베딩(의미검색 보조).
    - `research_graph` (Neo4j): 지식 그래프 — 노드/엣지로 관계·인과.
  - `agent`의 RAG(`GET /research/search`)는 내부에서 **그래프 traversal(Cypher) + 사실/벡터 회수**를 합쳐 근거를 반환. **agent↔research 계약은 불변**(엔진 변화는 research 내부).
- **그래프 모델(초안, /design 확정)**: 노드 `Stock`·`Event`·`Sector`·`Company/Person`; 엣지 `HAS_EVENT`·`AFFECTS`·`SUPPLIES`·`COMPETES`·`BELONGS_TO` 등.
- **진짜 일 = 추출 파이프라인**: 뉴스 → 엔티티(NER) + 관계추출 → 그래프 엣지. **여기가 품질의 원천.** `news-feed`(종목·사건 태깅) + `research`(관계 구축).
- **트레이드오프(수용)**: 새 datastore(운영·백업·Cypher 학습) 추가. 시계열 시세는 그래프 DB에 부적합 → **Postgres 병행(2-store)**. 관계 추론이 알파의 핵심이라 판단해 수용.
- **대안(기각)**: 관계형+pgvector만(관계 추론 약함 — 알파① 얕아짐). Apache AGE(단일 DB 장점이나 전용 그래프 대비 성능·도구 열위, 재작업 회피 목적상 전용 DB 직행).
- **관련**: [0002](pgvector는 보조로 유지), [0004](알파① 근거·정확). 반영: `doc/ref/architecture/00.아키텍처.html`·`01.research.md`.
