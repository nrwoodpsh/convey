# ADR 0004 — 제품 알파(차별점) 정의 · 아키텍처 무게중심

- **상태**: 채택 (2026-07-18) — 이후 모든 아키텍처·스코프 결정의 최상위 근거

> **갱신([0006])**: 아래 "pgvector 보조" 언급은 [0006]에서 **POC 제외·보류**로 갱신(RAG는 GraphRAG+SQL). 알파 4가지 정의 자체는 불변.
- **맥락**: "스크립트 한 줄 → 영상"은 이미 외부 생성형 툴(Sora·Runway·HeyGen 등)이 API 한 방으로 해주는 **커모디티**다. 우리 파이프라인의 절반(일반 스크립트 문장·broll 이미지·TTS·영상 스티칭)은 방어가능한 차별점이 없다. 따라서 **어디에 복잡도를 투자할지**를 알파부터 정의해야 한다. 도메인은 **주식/시장 콘텐츠**(한국어 쇼츠).

- **결정 — 알파 4가지에 투자, 나머지는 외주(얇게)**:
  1. **근거 있는 정확한 정보** — 주가·실적·이벤트를 환각 없이 **출처와 함께**. "안 틀리는 금융 콘텐츠". (RAG·구조화 저장·출처 보존의 진짜 이유)
  2. **이슈 종목 자동 선별** — "오늘 뭘 만들지"를 시세 급변 + 뉴스 빈도로 자동 포착. 편집적 신호.
  3. **정확한 차트·수치 렌더링** — 생성형 영상툴이 못하는 것: 정확한 종가·등락률·차트를 영상에 정확히 박기. 템플릿 렌더링을 우리가 통제.
  4. **자동화·양산 속도** — 일관 포맷으로 하루 N개 안정 양산.

- **커모디티(외부 API로 얇게, 알파 아님)**: 일반 스크립트 문장 생성, 장식용 broll 이미지, TTS 목소리, 영상 스티칭, YouTube 업로드.

- **아키텍처 귀결 (무게중심을 앞쪽으로)**:
  - `market-feed`는 버리는 원형이 아니라 **실제 시세 소스**(이슈감지·차트데이터).
  - `research-feed` → **`news-feed`**(수집이 아니라 **종목·사건 분류·태깅**).
  - `research`는 벡터 뭉텅이가 아니라 **관계형(종목·사건·기사) 뼈대 + pgvector 보조 + 출처**. ([0002] pgvector를 주력→보조로 격하)
  - **`issue-detector` 신규 서비스**(알파 2).
  - 미디어: broll·TTS = 외부 API 얇은 래퍼 / **`video-assembly`(차트·수치 렌더)만 우리 것**(알파 3).
  - **`llm-trainer`(LoRA) 제거** — 톤은 moat가 아님.

- **빌드 순서(의존성 순)**: ①데이터 기반(market-feed+news-feed+research 구조화) → ②issue-detector → ③근거 스크립트(agent 엄격 인용) → ④정확 렌더링(video-assembly) → ⑤양산 루프(스케줄·잡 재개·안정성). 커모디티는 아무 단계에나 얇게 삽입.

- **트레이드오프**: 커모디티를 외주하면 외부 API 의존·비용이 생기지만, 알파가 아닌 곳에 자원을 쓰지 않는다. 넷을 동시에 짓지 않고 순서로 관리(단일 사용자 규모).

- **관련**: [0001-multimedia-pipeline-pivot](멀티미디어 유지하되 커모디티/알파 구분 추가), [0002-pgvector-rag](pgvector 주력→보조 격하), [0003-external-media-api-guardrail], [0005-knowledge-graph-neo4j](알파① 지식 레이어를 Neo4j 그래프로).
