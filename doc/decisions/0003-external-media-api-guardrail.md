# ADR 0003 — 가드레일 정제: 외부 LLM 금지를 "텍스트 한정"으로 축소

- **상태**: 채택 (2026-07-18) — CLAUDE.md §4의 기존 "외부 LLM API 금지" 가드레일을 대체·정제함
- **맥락**: 기존 가드레일은 "외부 LLM API 호출 금지 — 로컬 Ollama만"으로 모든 외부 추론을 막았다. 그러나 멀티미디어 전환([0001])으로 이미지·TTS·영상클립을 외부 API로 생성하기로 결정 → 가드레일이 파이프라인과 충돌.
- **결정**: 금지 범위를 **텍스트 한정**으로 좁힌다.
  - **텍스트 LLM·RAG 임베딩**: 로컬 Ollama만 (리서치 원문 데이터 보호 유지).
  - **미디어 생성(이미지·TTS·영상클립)**: 외부 API 허용. 단 부패방지 계층(`external_client.py`)에 가두고, **원문 텍스트 전체를 외부로 넘기지 않음**(생성용 프롬프트·에셋만).
  - **발행**: YouTube Shorts 자동 업로드는 **사람 승인 후에만**(자동 발행 금지 유지).
- **트레이드오프**: 외부로 나가는 데이터 표면이 늘어남(프롬프트·에셋) vs 멀티미디어 생성 실현. → 원문 본문은 로컬에만, 외부엔 파생 프롬프트만.
- **영향**: CLAUDE.md §4 문구 교체, §6에 외부 API 자격증명 정책 추가. 미디어 워커 설계 시 부패방지 계층 필수.
- **대안(기각)**: 전면 로컬 유지(로컬 미디어 GPU) — [0001]에서 기각. 가드레일 완전 삭제 — 원문 유출 위험, 부적절.
- **관련**: [0001-multimedia-pipeline-pivot], [0002-pgvector-rag]. CLAUDE.md §4·§6 반영.
