# 20260723-task-ner.md

> 라운드 ㉚ (엔티티 인식 확대 — 개방형 NER + 수집 멱등 + 전량 백필). ADR 0014.
> 계약: `api-contract-ner.py`. research 도메인(지식 그래프·추출). 텍스트 LLM은 로컬 Ollama만.

## 1. Requirements

- **문제**: 그래프가 **46개 사전**에 갇혀 노드 33·관계 69(기사 1,085건 대비 빈약). 사전 밖 기업·인물·사건은 그래프에 안 들어와 GraphRAG가 얕다.
- **목표**: 로컬 Ollama로 기사에서 **엔티티+관계를 개방형으로 추출**(사전=고정확 seed와 합집)해 그래프를 채운다. 가드레일 유지 — 추출 엔티티는 **본문 실재(부분문자열)**, 관계는 **근거(article_id)·엣지 제한**(환각 0).
- **부수 과제(중요)**: news-feed가 매 폴링 사이클 같은 기사를 재발행 → research 중복 저장·재추출. NER는 비싸므로 **수집 멱등**(source_url 기준 skip)이 전제.
- **Acceptance Criteria**:
  - [x] AC1: `extract_graph(text, seed, llm)` — Ollama 1콜 `{entities, relations}`, 본문 substring 검증·정규화·스톱워드·엣지 제한. **검증: 단위테스트 6(환각컷·엣지·스톱워드·seed).**
  - [x] AC2: research consumer가 신규 기사에 `extract_graph` 적용 → 사전 밖 엔티티/관계 그래프 생성. **검증: 사전 밖 인물 "젠슨황" 노드, 노드 33→36+.**
  - [x] AC3: **수집 멱등**(source_url) — 재발행돼도 재저장·NER skip. **검증: 기사 수 1145 안정(폴링 재발행에도 불변).**
  - [x] AC4: 백필(`--llm`) `extract_graph` 사용, 재개 가능. **검증: 소량 백필로 노드·관계 증가. 전량 1,085는 사람 실행(부하).**
  - [x] AC5: 가드레일 불변 — 관계 근거(source_article_id) 결속·엣지 제한, 수치 생성 0, 로컬 Ollama만.

## 2. 사각지대 & 핵심 결정 (수정 가능성 순)

- **사각지대**:
  - **LLM 부하**: NER는 기사당 Ollama 1콜. 폴링마다 전 기사 재추출하면 폭증 → **수집 멱등이 필수 전제**(신규만 추출). 라이브는 신규 기사만, 백필은 일회성 전량.
  - **환각 엔티티**: LLM이 본문에 없는 엔티티를 만들 수 있음 → **부분문자열 검증**으로 폐기. 관계 subject/object도 검증 엔티티∪seed로 제한.
  - **정규화 부재 시 중복 노드**: "삼성전자"/"삼성"/"삼성전자(주)" 분리 → 정규화(공백·괄호·조사 정리 + seed 별칭 매핑 + dedup). 완전한 개체 연결(entity linking)은 후속.
  - **노이즈 노드**: "정부·시장·기업" 등 일반어가 엔티티로 → `ENTITY_STOPWORDS`·`ENTITY_MIN_LEN` 필터.
  - **기존 중복 기사**: 멱등 도입 전 쌓인 중복 기사(1,085에 중복 포함 가능) — 백필은 그대로 두되, 멱등은 이후 신규에 적용. (기존 정리는 별도.)
  - **파싱 취약성**: LLM JSON이 설명·<think> 감쌀 수 있음 → 관대한 JSON 배열/객체 추출(기존 `_parse` 패턴 재사용).
- **핵심 결정**(택함 + 기각):
  - **결정 1 — NER 방식**: 택함 = **로컬 Ollama 개방형 추출(엔티티+관계 1콜)** / 기각 = 통계 NER(spaCy/KoNLPy — 새 ML 의존·다운로드·한국어 품질편차) / 사유 = 기존 llm-inference 재사용·가드레일(로컬 LLM)·1콜 효율. (사용자 확정) ADR 0014.
  - **결정 2 — 추출 위치**: 택함 = **research consumer**(이미 LLM 호출·그래프 소유) — news-feed 사전 seed는 그대로 넘기고 research가 개방 추출로 확장 / 기각 = news-feed에 LLM 추가(현재 LLM 미사용·결합↑) / 사유 = 그래프·LLM은 research 책임.
  - **결정 3 — 라이브 게이팅**: 택함 = **신규 기사(멱등)만 추출**, 종목 seed 유무 무관하게 개방 추출 / 기각 = 매 폴링 전량(부하 폭증) / 사유 = 멱등으로 1회만, 커버리지 최대.
  - **결정 4 — 백필 범위**: 택함 = **전량 1,085**(사용자 확정) / 기각 = 태깅분만 / 사유 = 커버리지 최대(일회성·재개).
  - **결정 5 — 엔티티 검증**: 택함 = **본문 부분문자열 필수 + 정규화 + 스톱워드** / 기각 = LLM 출력 신뢰 / 사유 = 환각·노이즈 차단(알파1).

## 3. UI/UX

- 사용자 화면 변화 없음(백엔드 그래프 품질). 결과는 시나리오의 관계 문장·GraphRAG 회수가 풍부해짐(대시보드·영상에 간접 반영).

## 4. Logic

- **extract 모듈**(`research/app/extract/`): `extract_graph(text, seed_entities, llm) -> ExtractedGraph`.
  - 프롬프트: "기사에서 엔티티(기업·인물·사건·기관·섹터)와 그들 간 관계를 JSON으로. 엔티티는 본문에 나온 것만. 엣지는 {EDGE_TYPES}만. 설명 없이 JSON." (기존 `build_prompt`/`_parse` 확장.)
  - 후처리: 엔티티 = 본문 substring 검증 → 정규화(strip·괄호·조사, seed 별칭 매핑) → 스톱워드·min-len 필터 → dedup. 관계 = subject/object ∈ (검증 엔티티 ∪ seed) ∧ edge ∈ EDGE_TYPES.
  - `extract_relations`(기존)는 유지(호환) 또는 `extract_graph`가 대체.
- **research consumer `handle_ingested`**:
  - **멱등**: source_url이 이미 있으면 저장·추출 skip(신규만 진행). (기존: 무조건 add → 중복.)
  - 신규 기사: Article 저장 → `extract_graph(body, seed=event entities, llm)` → 엔티티 노드 + 관계 upsert(근거=article.id). 기존 결정론 종목→섹터/사건 엣지 유지.
- **backfill**(`app/backfill.py --llm`): 전량(1,085) 기사에 `extract_graph` 재실행(진행 로그·재개). 사람 실행.
- **정규화 헬퍼**: `common.stocks` 별칭 + 간단 규칙. 완전 entity linking은 후속.

## 5. Implementation Split (다음 /builder)

- **research**: `extract/relations.py`(또는 신규 `extract/graph.py`)에 `extract_graph` + 정규화·검증. `consumer.py` 멱등 + extract_graph 연결. `backfill.py` extract_graph 사용.
- (선택) `common.stocks` 별칭 맵 소량.

## 7. Verification (다음 /builder)

- extract_graph 단위: 스텁 LLM으로 엔티티 substring 검증·엣지 필터·정규화 확인(환각 엔티티 폐기).
- 멱등: 같은 source_url 2회 → articles 행·그래프 증가 0.
- 라이브: 신규 기사 1건 → 사전 밖 엔티티(예: 인물·비사전 기업) 노드 생성 확인.
- 백필 소량(--limit) → 관계 증가; 전량은 사람 실행(부하). 그래프 노드/관계 전후 카운트.
- 가드레일: 무출처 관계 0(source_url), 로컬 Ollama만. mypy·계약·단위테스트 통과.
