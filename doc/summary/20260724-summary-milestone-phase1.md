# 20260724 — 마일스톤: CONVEY 1차 개발 완료 (로컬 전자동 파이프라인)

> **한 줄 결론**: 로컬 도커 스택에서 **주식 리서치 → 유튜브 쇼츠(mp4)**가 사람 개입 없이 전자동으로 만들어진다. 알파 4개가 모두 코드로 구현·검증됐다. 외부 키·클라우드가 필요한 발행·인증·배포(C)는 **의도적으로 이연**한다.
> 작업자: park seung hyun · 날짜: 2026-07-24 · 브랜치: main · 기준: 로컬 `docker compose` 스택.

---

## 1. 무엇을 1차 완료로 보는가 (범위 정의)

**1차 = 로컬에서 전자동 파이프라인이 완성본을 만드는 것.** 판매·배포·외부 발행은 2차(C)로 분리한다.

```
[뉴스·시세]  →  research(그래프·NER)  →  issue-detector(이슈 선별)
     →  content(스크립트·시나리오)  →  video-assembly(ffmpeg 합성)  →  완성본 mp4
     →  (사람 승인)  →  YouTube 발행 [C — 이연]
```

- **되는 것**: 수집 → 그래프 저장 → 이슈 선별 → 대본 → 음성·자막·차트·배경 합성 → 쇼츠 mp4 + 미리보기까지 **무인**.
- **경계**: 최종 발행(YouTube 업로드)은 **사람 승인 후에만** — 자동은 완성본 생성까지(가드레일 일치).

## 2. 알파(차별점) 구현 현황

| 알파 | 상태 | 구현 근거(코드·라운드) |
|:--|:--|:--|
| ① 근거 있는 정확한 정보(출처·환각금지) | ✅ | GraphRAG(Neo4j Cypher + Postgres SQL), 관계마다 `source_article_id` 결속, 개방형 NER의 본문 substring 환각컷 (라운드⑤·㉕·㉚) |
| ② 이슈 종목 자동 선별 | ✅ | `issue-detector` → `issue.selected` → 자동 양산 (라운드⑬) |
| ③ 정확한 차트·수치 렌더링 | ✅ | matplotlib 한글 차트 + 수치 오버레이(팝·금색 강조), 수치는 데이터에서만(LLM 생성 금지 가드) (라운드㉑·㉖) |
| ④ 자동화·양산 | ✅ | 이슈→제작 무인 루프, 자동 양산 중복 회피, 구간 TTS 병렬 (라운드⑬·㉘·㉙) |

> 커모디티(일반 문장·broll·TTS·스티칭)는 외부 API로 얇게(edge-tts·Pexels), 영상 합성만 로컬 ffmpeg — 설계 의도대로.

## 3. 구성 (서비스·데이터)

- **도메인 3개**: `research`(시세·뉴스 → 관계 그래프+사실) · `content`(스크립트·미디어 자산·완성본) · `publishing`(발행).
- **서비스**: gateway · research · news-feed · market-feed · issue-detector · agent · content · video-assembly · publishing · llm-inference (+ 인프라: kafka · kafka-ui · postgres · neo4j).
- **데이터 분업**: Neo4j(종목·사건의 **관계**) / Postgres(기사·시세 **사실**). RAG는 GraphRAG+SQL, 벡터 임베딩 제외(ADR 0006).
- **텍스트 LLM**: 로컬 Ollama만(원문 보호 가드레일). 미디어 생성만 외부 API 허용(부패방지 계층 경유).

## 7. 품질에 대한 정직한 현주소 (다음 라운드 입력)

기능은 완성이나 **"제품 출시 품질"은 아직 확정 아님**. 핵심 변수는 **그래프 데이터의 두께**다.

- 그래프가 얇으면 영상이 "무슨 섹터다" 수준에 그치고, 두꺼우면 "왜 올랐나(인과 연쇄)"를 설명할 수 있다 → **백필 완료가 품질의 전제**.
- 백필 완료 후 **동일 종목 재생성으로 품질 재평가** 권장(대본의 인과 문장·관계 회수량 비교).
- 후속 품질 후보: entity linking(동의어 병합 — "삼성/삼성전자")·엔티티 타입 라벨(:Person·:Event)·긴 기사 LLM 타임아웃 완화(본문 앞부분 절삭).
